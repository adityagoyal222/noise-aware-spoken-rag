# Thesis Project Instructions

## Project summary
This project is a thesis pipeline for speaker-attributed candidate selection from audio transcripts.

Core pipeline:
- ASR: Whisper.
- Diarization: pyannote.
- Base chunks: Whisper ASR segments.
- Alignment: diarization is aligned to ASR by time overlap, not by exact timestamps.
- Do not silently collapse all multi-speaker ASR segments into one speaker.
- If one speaker dominates, assign it.
- If a chunk is mixed, either split it or keep it and penalize it numerically.
- Keep DiarStab and MixPenalty separate.
- Final ranking formula:
  R(c) = a*s(q,c) + b*ASRConf(c) + g*DiarStab(c) + d*TurnComp(c) + e*Redund(c) - m*MixPenalty(c)

## Scripts (pipeline stages)
Prefer scripts over notebooks for anything that runs repeatedly or across configurations.
- `scripts/run_asr.py` — Run Whisper ASR for all model sizes. Args: `--models`, `--retry-failed`.
- `scripts/run_alignment.py` — Align ASR output with diarization. Args: `--model <name>` or `--all`.
- `scripts/generate_eval_queries_new.py` — Generate eval queries with relevance labels and reference answers.
- `scripts/retrieval_dense.py` — Dense cosine-similarity retrieval baseline. Args: `--model`, `--topk`.
- `scripts/retrieval_bm25.py` — (not yet created) BM25 retrieval.
- `scripts/retrieval_crossencoder.py` — (not yet created) Cross-encoder reranker.
- `scripts/retrieval_naes_h.py` — (not yet created) Heuristic noise-aware reranker.
- `scripts/retrieval_naes_l.py` — (not yet created) Learned noise-aware reranker.

## Current thesis goals
1. Finish and maintain alignment outputs.
2. Build and evaluate all retrieval pipeline variants.
3. Evaluate retrieval and answer quality with labeled queries and reference answers.
4. Support literature review and thesis writing.

## Output directories
Use only these directories for project outputs:
- `data/amicorpus/` = the full dataset with 100 hours of audio. DO NOT LOOK FOR THIS DIRECTORY OR SEARCH INTO IT.
- `data/raw_audio/` = actual subset of audio that is going to be used in the experimentation. DO NOT LOOK FOR THIS DIRECTORY OR SEARCH INTO IT.
- `data/gold_transcripts` = transcripts for all the files within the `data/raw_audio` folder. This was provided by the amicorpus
- `data/asr_outputs/<model>/` = ASR outputs per Whisper model size (tiny, base, small, medium, large-v3). Each subdirectory has raw/, segments/, transcripts/, checkpoints/, logs/.
- `data/diarization_outputs` = Output generated by pyannote for diarization. Done once, not re-run.
- `data/eval` = includes all the files related to the queries that will be used as the baseline for evaluation.
- `data/aligned_chunks/<model>/` = aligned ASR-diarization chunk CSVs per model.
- `data/analysis` = simple analysis of the data that is going to be used.
- `data/metrics/<model>/` = per-model feature tables (e.g. `medium/`, `large-v3/`). Embedding cache lives at `data/metrics/chunk_embeddings_minilm_<model>.npz`.
- `data/retrieval_results/<pipeline>/` = retrieval outputs per pipeline (dense/, bm25/, crossencoder/, naes_h/, naes_l/). Each pipeline directory holds results, per-query metrics, and summary CSVs.
- `notebooks/` = exploratory notebooks only. Do not use for pipeline stages that run repeatedly.
- `scripts/` = all runnable pipeline scripts.
