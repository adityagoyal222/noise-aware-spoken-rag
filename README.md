# Noise-Aware Spoken RAG 🎙️

**Master's thesis research** — a confidence-aware reranking method that recovers retrieval and answer quality in spoken RAG pipelines, even when automatic speech recognition (ASR) and speaker diarization are noisy.

## The problem

Spoken RAG systems cascade three error-prone stages — transcription, diarization, retrieval — and errors compound. Most retrieval methods treat transcripts as if they were clean text, ignoring the fact that some chunks are backed by high-confidence transcription and stable speaker attribution, while others are noisy guesses. This thesis asks: **can we use the ASR and diarization system's own confidence signals to make retrieval robust to this noise, without retraining any model?**

## Research questions

1. **RQ1** — How do word error rate (WER) and diarization error rate (DER), independently and combined, degrade retrieval and answer quality in a cascaded spoken RAG pipeline?
2. **RQ2** — Can a confidence-aware reranker that uses ASR confidence, diarization stability, and speaker-turn signals outperform baselines that ignore noise entirely?
3. **RQ3** — What is the individual contribution of each noise-aware signal (via feature ablation)?
4. **RQ4** — Does the benefit of confidence-aware reranking scale consistently across noise levels, or are there diminishing returns?

## Approach

A modular, **retraining-free reranking layer** sits on top of standard dense retrieval, scoring each candidate chunk as:

```
R(c) = α·s(q,c) + β·ASRConf(c) + γ·DiarStab(c) + δ·TurnComp(c) − ε·Redund(c) − μ·MixPenalty(c)
```

where `s(q,c)` is semantic similarity and the remaining terms are audio-derived quality signals:

- **ASRConf** — normalized transcription confidence (from Whisper's average token log-probability)
- **DiarStab** — composite diarization stability/consistency score
- **TurnComp** — utterance/turn completeness
- **Redund** — repeated-content penalty (Jaccard similarity)
- **Purity / MixPenalty** — single-speaker concentration vs. speaker-mixing penalty

Two reranker variants are implemented and compared: **NAES-H** (heuristic, hand-weighted) and **NAES-L** (learned, via logistic regression with meeting-stratified 5-fold cross-validation).

## Pipeline

| Stage | Tool |
|---|---|
| Transcription | `faster-whisper` (5 model sizes: tiny → large-v3, CPU int8) |
| Speaker diarization | `pyannote.audio` v3.x |
| Dense retrieval | MiniLM embeddings + FAISS |
| Lexical baseline | BM25 |
| Reranking | Cross-encoder (`ms-marco-MiniLM-L-6-v2`), NAES-H, NAES-L |
| Upper bound | Oracle retrieval on gold transcripts |

**Dataset:** a curated ~36-meeting subset of the **AMI Meeting Corpus**, filtered for audio quality, 3–5 speakers, and sufficient duration — using mixed-channel (not per-speaker headset) audio to reflect realistic deployment conditions.

**Evaluation set construction:** quality-gated chunks (`ASRConf > 0.35`, `TurnComp > 0.3`, `MixPenalty < 0.6`, >8 words), with queries generated by local LLM inference over meeting sections, relevance labels combining LLM judgment with semantic cross-window recovery, cross-encoder re-scoring, and NLI faithfulness verification.

**Metrics:** retrieval quality via NDCG@5, MRR, Recall@K; answer quality via Exact Match, F1, and BERTScore; results stratified by WER/DER tier, meeting length, speaker count, and query difficulty.

## Key findings so far

- Only **Whisper `medium`** produces usable transcripts on this data (WER≈0.41) — smaller models hallucinate severely (WER≈0.89), and `large-v3` paradoxically performs *worst* (WER≈1.71), making retrieval infeasible at those settings.
- **Evaluation scope matters enormously**: a global (cross-meeting) index lets audio-quality proxies cause "rank pollution," promoting chunks from the wrong meeting. Restricting to **per-meeting retrieval** raised Cross-Encoder NDCG@10 from 0.464 → 0.757 and NAES-L from 0.278 → 0.624.
- **Feature ablation** (per-meeting scope): semantic similarity still dominates (−0.118 NDCG if removed), but noise-aware signals add real, complementary value — MixPenalty (−0.016), TurnComp (−0.011), and DiarStab (−0.009) all help; ASRConf is the one signal that's *actively harmful* within-meeting (+0.032 when dropped).

## Status

Retrieval systems and ablations complete on the Whisper-medium configuration. Answer generation and final human-reviewed relevance labels are in progress. Benchmarked against BM25, dense, and cross-encoder baselines, with the MEETINGQA benchmark integration as deferred/stretch scope.

## Scope & limitations

Inference-only (no neural encoder training, due to hardware constraints); binary relevance labels are primary, graded labels are a stretch goal; manual review queue generated but not yet fully processed.
